04-C++-组合与继承

发表于2022-08-05|更新于2022-08-07|C++
|字数总计:1.7k|阅读时长:6分钟|阅读量:

类和类之间的关系包含以下三种：

  • Composition - 复合
  • Delegation - 委托
  • Inheritance - 继承

Composition(复合)

复合表示类中有一个成员变量(非指针变量), 即表示has-a, 如下图所示:类queue中有一个deque(has a); 变量和类本身的寿命是同生同灭

Adapter概念

deque是两端都可以进出的双队列, 而queue是先进的队列,明显deque功能比queue强大;上面的queue并没有重新创建新的函数都是对deque进行了一层包装,这就是适配器设计模式

委托表示类中有一个成员变量(指针变量),即Composition by reference 如下图所示:类String中有一个StringRep, String是Handle, StringRep是Body； 指针变量指向的Body和Handle的生命周期不一致(可能Handle对象先被创建, Body之后用到才被创建)

其中n是用作引用计数

pimpl

像上述的设计模式即为pimpl(Pointer to Implementation), Handle部分被用户直接调用, 实现部分由Body完成, 这种手法被称为编译防火墙(Handle不用重新编译, 单独编译Body部分即可)




Inheritance(继承)

继承表示类在数据包含了父类的数据部分, 即表现为 is a; 但这不是继承的核心要义, 继承和虚函数搭配才是核心

基类析构函数

如果一个类有可能成为基类, 则应该把该类的析构函数定义为virtual(理由比较长),这是一个良好的编程习惯, 否则会出现undefined behavior

虚函数

继承搭配虚函数时(Inheritance with virtual functions),才能产生强有力的效果, 类中一个函数被定义为virtual时,是希望子类重新定义该函数

  • non-virtual函数: 你不希望derived class 重新定义override该函数

  • virtual函数: 你希望derived class重新定义override该函数,而且你已经对该函数有一个默认定义了

  • pure virtual函数: 你希望derived class 一定要重新定义override该函数,而且你已经对该函数有没有默认定义(其实纯虚函数可以有定义)

上图是图像绘制的一个示例:

  • 基类Shape中分别定义了非虚函数(objectID和plot), 虚函数(error)和纯虚函数(draw)
    • 纯虚函数必须 在派生类中重新定义, 否则报错error: invalid cast to abstract class type 'learn::Rectangle'
    • 不是纯虚函数必须 在基类必须定义,否则报错undefined reference tovtable for learn::Shape’`
    • 派生类中定义的虚函数声明时可以添加virtual, 但是在类的外部定义该函数时不能添加virtual, 否则报错
  • rect对象调用了基类中的plot方法, 实际上是Shape::plot(&rect)
  • 基类中的plot方法中调用了draw方法, 实际上this->draw(pos); => (*(this->vptr)[n])(this, pos)
  • 基类代码可能早已写好, 将draw方法写成pure virtual方法, 而派生类之后在定义了该方法(基类不知道如何实现该方法), 这种设计模式是Template Method
