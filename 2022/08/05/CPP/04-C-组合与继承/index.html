04-C++-组合与继承

发表于2022-08-05|更新于2022-08-16|C++
|字数总计:2.5k|阅读时长:10分钟|阅读量:

类和类之间的关系包含以下三种：

  • Composition - 复合
  • Delegation - 委托
  • Inheritance - 继承

Composition(复合)

复合表示类中有一个成员变量(非指针变量), 即表示has-a, 如下图所示:类queue中有一个deque(has a); 变量和类本身的寿命是同生同灭

#ifndef _QUEUE_H_
#define _QUEUE_H_
#include <iostream>

namespace learn{

    template<class T>
    struct Itr
    {
        T* cur;
        T* first;
        T* last;
        T** node;
    };

    template <class T>
    class deque
    {
        public:
            deque(){ std::cout << "deque ctor." << std::endl;}
            ~deque(){ std::cout << "deque dctor." << std::endl;}
        protected:
            Itr<T> start;
            Itr<T> finish;
            T** map;
            unsigned int map_size;
    };

    template <class T, class Sequence = deque<T>>
    class queue{
        protected:
            Sequence c;// 复合 has-a
        public:
            queue(){ std::cout << "queue ctor." << std::endl; }
            ~queue(){ std::cout << "queue dctor." << std::endl; }
            bool empty() const {return c.empty();}
            size_t size() const {return c.size();}
            
            void push(const T& x){c.push_back(x);}
            void pop(){c.pop_front();}
    };
}
#endif

Adapter概念

deque是两端都可以进出的双队列, 而queue是先进的队列,明显deque功能比queue强大;上面的queue并没有重新创建新的函数都是对deque进行了一层包装,这就是适配器设计模式

构造由内而外

Container的构造函数先调用Component的default 构造函数, 然后执行自己的构造函数,即:

Container::Container(...): Component() {...};

析构由外而内

Container的析构函数先执行自己的析构任务, 然后Component的析构函数,即:

Container::~Container(...): { ...; ~Component()};




Delegation(委托)

委托表示类中有一个成员变量(指针变量),即Composition by reference 如下图所示:类String中有一个StringRep, String是Handle, StringRep是Body； 指针变量指向的Body和Handle的生命周期不一致(可能Handle对象先被创建, Body之后用到才被创建)

其中n是用作引用计数

pimpl

像上述的设计模式即为pimpl(Pointer to Implementation), Handle部分被用户直接调用, 实现部分由Body完成, 这种手法被称为编译防火墙(Handle不用重新编译, 单独编译Body部分即可)

#ifndef _MYSTRINGDELEGATION_H_
#define _MYSTRINGDELEGATION_H_

namespace learn
{
    class StringDe;
    class StringRep{
        friend class StringDe;
        StringRep(const char* s);
        ~StringRep();
        int count;// 引用计数器
        char *rep;
    };
    
    class StringDe{
        public:
            StringDe();
            StringDe(const char *s);
            StringDe(const StringDe& s);
            StringDe& operator=(const StringDe& s);
            ~StringDe();
        private:
            StringReq* rep; // pimpl - Pointer to Implementation
    };
}   
#endif




Inheritance(继承)

继承表示类在数据包含了父类的数据部分, 即表现为 is a; 但这不是继承的核心要义, 继承和虚函数搭配才是核心

#ifndef _LIST_NODE_H_
#define _LIST_NODE_H_
#include <iostream>
namespace learn
{
    struct _List_node_base
    {
        _List_node_base(){std::cout << "_List_node_base ctor."<< std::endl;}
        virtual ~_List_node_base(){std::cout << "_List_node_base dctor."<< std::endl;}
        _List_node_base *_M_next;
        _List_node_base *_M_prev;
    };
    
    template<typename _Tp>
    struct _List_node: public _List_node_base
    {
        _List_node(){std::cout << "_List_node ctor."<< std::endl;}
        ~_List_node(){std::cout << "_List_node dctor."<< std::endl;}

        _Tp _M_data;
    };
}

#endif

构造由内而外

Derive的构造函数先调用Base的default 构造函数, 然后执行自己的构造函数,即:

Derive::Derive(...): Base() {...};


析构由外而内

Derive的析构函数先执行自己的析构任务, 然后Base的析构函数,即:

Derive::~Derive(...): { ...; ~Base()};


基类析构函数

如果一个类有可能成为基类, 则应该把该类的析构函数定义为virtual(理由比较长),这是一个良好的编程习惯, 否则会出现undefined behavior

虚函数

继承搭配虚函数时(Inheritance with virtual functions),才能产生强有力的效果, 类中一个函数被定义为virtual时,是希望子类重新定义该函数

  • non-virtual函数: 你不希望derived class 重新定义override该函数

  • virtual函数: 你希望derived class重新定义override该函数,而且你已经对该函数有一个默认定义了

  • pure virtual函数: 你希望derived class 一定要重新定义override该函数,而且你已经对该函数有没有默认定义(其实纯虚函数可以有定义)

上图是图像绘制的一个示例:

  • 基类Shape中分别定义了非虚函数(objectID和plot), 虚函数(error)和纯虚函数(draw)
    • 纯虚函数必须 在派生类中重新定义, 否则报错error: invalid cast to abstract class type 'learn::Rectangle'
    • 不是纯虚函数必须 在基类必须定义,否则报错undefined reference tovtable for learn::Shape’`
    • 派生类中定义的虚函数声明时可以添加virtual, 但是在类的外部定义该函数时不能添加virtual, 否则报错
  • rect对象调用了基类中的plot方法, 实际上是Shape::plot(&rect)
  • 基类中的plot方法中调用了draw方法, 实际上this->draw(pos); => (*(this->vptr)[n])(this, pos)
  • 基类代码可能早已写好, 将draw方法写成pure virtual方法, 而派生类之后在定义了该方法(基类不知道如何实现该方法), 这种设计模式是Template Method

文件系统-File System

文件系统中包含文件和目录, 目录中包含文件和目录; 那么这个问题可以被抽象为:

  • 对象A是最基本的(Primitive)
  • 对象B(Composite)是可以包含对象A和对象B

那么这种场景可以使用如下设计模式-Composite

实现tree函数

#include "fileSystem.h"
#include <iostream>
int main(int argc, char **argv)
{
    learn::Primitive fileSystem_h("fileSystem.h");
    learn::Primitive fileSystem_cpp("fileSystem.cpp");
    learn::Primitive CMakeLists_txt("CMakeLists.txt");

    learn::Composite fileSystem_dir("fileSystem");

    fileSystem_dir.add(&fileSystem_h);
    fileSystem_dir.add(&fileSystem_cpp);
    fileSystem_dir.add(&CMakeLists_txt);

    // ------------------------------------------------
    learn::Primitive fileSystem_main_cpp("fileSystem_main.cpp");

    learn::Composite learn_dir("00-learn");

    learn_dir.add(&fileSystem_dir);
    learn_dir.add(&fileSystem_main_cpp);

    learn_dir.tree();
    return 0;
}

fileSystem.h的实现过程

  • 其中Component的add不能定义为纯虚函数, 因为根据需要文件没有子级
#ifndef _FILESYSTEM_H_
#define _FILESYSTEM_H_

#include <vector>
#include <iostream>
#include <string>

namespace learn
{
    class Component
    {
    public:
        Component(const char *name) : m_name(name) {}
        virtual void add(Component *c) {}
        virtual void tree(const int level = -1, const int curLevel = 1) const
        {
            std::cout << " - " << m_name << std::endl;
        }

        const std::string &name() const { return m_name; }

    private:
        std::string m_name;
    };

    class Primitive : public Component
    {
    public:
        Primitive(const char *name) : Component(name) {}
    };

    class Composite : public Component
    {
    public:
        Composite(const char *name) : Component(name) {}
        virtual void add(Component *c)
        {
            if (NULL == c){ return; }
            m_c.push_back(c);
        }

        virtual void tree(const int level = -1, const int curLevel = 1) const
        {
            // 文件夹
            std::cout << " d " << name() << std::endl;

            if (-1 != level && curLevel > level)
            { // 如果不要打印文件夹的子级文件, 这直接返回
                return;
            }
            const int nextLevel = curLevel + 1;
            // 文件夹下的文件或者目录
            for (size_t i = 0; i < m_c.size(); i++)
            {
                for (size_t j = 0; j < curLevel; j++)
                {
                    std::cout << "    ";
                }
                m_c[i]->tree(level, nextLevel);
            }
        }

    private:
        std::vector<Component *> m_c;
    };

}

#endif


Prototype

如果3年前,A公司设计了基类, B公司把A公司设计的框架买过去, 基于基类派生了一些列派生类; 3年前A公司在基类中需要创建B公司派生类对象

实现如上场景的关键在于:

  • 在所有派生类(LandSatImage和SpotImage)中定义一个本身类型的静态成员变量
    • 定义一个私有的默认不带参数的构造函数, 该函数的功能就是把静态的派生类对象注册到父类数组中
  • 同时基类中定义一个纯虚函数(clone), 派生类重新定义clone函数, clone函数的功能就是通过静态派生类对象new一个派生类对象

为什么不把clone函数写成静态成员函数, 但是通过静态成员函数必须事先直到class name
