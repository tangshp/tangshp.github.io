基本命令

调试代码

生成调试信息

进入调试模式

  • gdb --args executable arg1 arg2 进入调试模式
  • run指令会一直执行程序, 直到执行结束。 如果程序中手动设置有断点, 则run指令会执行程序至第一个断点处；
  • start 指令会执行程序至main()主函数的起始位置，即在main()函数的第一行语句处停止执行（该行代码尚未执行）。
  • print 指令可以查看变量状态

退出调试模式

带参数

  • gdb --args executable arg1 arg2 进入调试模式
  • 可以在进入调试模式后, 使用r arg1 arg2加入参数
  • 可以在进入调试模式后, 使用start arg1 arg2加入参数

listsize

  • listsize命令可以显示源码信息
  • 设置list显示的行数

next

单步调试:单步指令中执行后的输出结果为是程序执行的输出和下一条执行语句

break

添加断点: break linenum 或者 break function 或者 break file
显示断电: info b

step

跳转到函数的第一条语句

continue

继续运行到下一个断电

where

查看当前在哪
