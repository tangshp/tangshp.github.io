Raspberry+Docker+Gitlab+Jenkins

发表于2022-06-05|更新于2022-06-12|搭建CI环境

docker

安装docker

  • 1、安装curl

    sudo apt install curl

  • 2、使用curl安装docker

    sudo curl -sSL https://get.docker.com | sh

  • 3、安装docker-compose

    sudo pip3 install docker-compose

docker的基本操作

  • 查看当前有哪些docker镜像
    ➜  ~ sudo -i
[sudo] password for tangshp:
root@tangshp-desktop:~# docker images
REPOSITORY               TAG       IMAGE ID       CREATED      SIZE
yrzr/gitlab-ce-arm64v8   latest    6db1d4ba809d   3 days ago   2.46GB

  • 查看当前哪些镜像被运行

    root@tangshp-desktop:~# docker ps -a
CONTAINER ID   IMAGE                           COMMAND             CREATED        STATUS                  PORTS                                                                                                         NAMES
e39d62cc5e81   yrzr/gitlab-ce-arm64v8:latest   "/assets/wrapper"   25 hours ago   Up 13 hours (healthy)   0.0.0.0:22->22/tcp, :::22->22/tcp, 0.0.0.0:80->80/tcp, :::80->80/tcp, 0.0.0.0:443->443/tcp, :::443->443/tcp   gitlab

  • 暂停指定正运行的镜像

    docker stop e39d62cc5e81

  • 将暂停的镜像删除

    docker rm e39d62cc5e81

  • 删除镜像

    docker rmi 6db1d4ba809d

  • 进入docker的bash

    ➜  ~ sudo docker exec -it gitlab bash

gitlab镜像

yml文件配置

保证在/opt/gitlab有config,data,logs文件夹与docker-compose.yml文件

➜  gitlab ls -l
total 16
drwxrwxr-x  3 root root 4096 6月   4 19:09 config
drwxr-xr-x 21 root root 4096 6月   4 19:41 data
-rw-r--r--  1 root root  710 6月   4 19:29 docker-compose.yml
drwxr-xr-x 21 root root 4096 6月   4 19:36 logs

docker-compose.yml文件内容如下
version: '3'
services:
  gitlab:
    image: yrzr/gitlab-ce-arm64v8:latest
    container_name: gitlab
    restart: always
    privileged: true
    environment:
      TZ: 'Asia/Shanghai'
      GITLAB_OMNIBUS_CONFIG: |
        external_url = "http://192.168.0.100"
        gitlab_rails['time_zone'] = 'Asia/Shanghai'
        gitlab_rails['smtp_enable'] = true
        gitlab_rails['gitlab_shell_ssh_port'] = 22
    ports:
      - '80:80'
      - '443:443'
      - '22:22'
    volumes:
      - '/opt/gitlab/config:/etc/gitlab'
      - '/opt/gitlab/logs:/var/log/gitlab'
      - '/opt/gitlab/data:/var/opt/gitlab'
    logging:
      driver: "json-file"
      options:
        max-size: "20m"
        max-file: "10"

运行gitlab镜像

  • 修改ssh端口
vim /etc/ssh/sshd_config

#Port 22
Port 5000
# 在/opt/gitlab路径下运行 以sudo运行
docker-compose up -d

docker-compose logs -f

  • 重启gitlab服务

jenkins

安装jenkins

  • 1、安装daemon
    sudo apt install daemon

  • 2、安装jenkins

    sudo apt install jenkins

  • 3、修改/etc/default/jenkins

    # user and group to be invoked as (default to jenkins)
JENKINS_USER=root
JENKINS_GROUP=root

  • 4、修改/var/lib/jenkins/hudson.model.UpdateCenter.xml

    http://mirror.xmission.com/jenkins/updates/update-center.json

  • 5、重启服务

    systemctl restart jenkins
  • 6、访问http://192.168.5.48:8080/

jenkins插件路径/var/lib/jenkins/plugins/

jenkins服务

systemctl start jenkins
systemctl restart jenkins
systemctl stop jenkins

jenkins的基本使用

jenkins集成gitlab

  • http
    • 设置凭据，在Manage Jenkins/Security/Manage Credentials/设置

    • 生成流水线脚本,在cubli_mini_master/配置/流水线语法/片段生成器中中生成代码片段
pipeline {
  agent {
      label 'ubuntu18.04'
    }
    stages{
        stage('检测环境')
        {
            steps{
                echo 'check the environment'
                sh '''
                java -version
                echo ""
                gcc -v
                echo ""
                cmake -version
                echo ""
                git version
                echo ""
                docker -v
                '''
            }
        }
        stage('拉取代码')
        {
            steps{
                echo 'git pull code from gitlab'
                checkout([$class: 'GitSCM', branches: [[name: '*/main']], extensions: [], userRemoteConfigs: [[credentialsId: 'e3704e36-e832-4786-8b3c-5c3fd90f7d59', url: 'http://192.168.5.48/cubli_mini_serial/cubli_mini_master.git']]])
            }
        }
        stage('编译构建')
        {
            steps{
                echo 'build the project'
            }
        }
        stage('项目部署')
        {
            steps{
                echo 'deploy the project'
            }
        }
    }
}
