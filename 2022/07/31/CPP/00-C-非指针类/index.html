00-C++-非指针类

发表于2022-07-31|更新于2022-08-16|C++
|字数总计:2.2k|阅读时长:9分钟|阅读量:

C++类可以分为两种类型:

  • 非指针类型的Class
  • 指针类型的Class

这两种类型的Class的重要区别在于拷贝构造、拷贝赋值和析构函数(Big Three)上的巨大差异, 主要在于编译器在拷贝时是傻瓜式对象内存拷贝:

  • 非指针类型的Class使用编译器给的拷贝构造、拷贝赋值和析构函数即可
  • 指针类型的Class必须自己实现这三个函数

本笔记对应的代码笔记链接-C++面向对象编程上, 对应complex的实现

非指针类Complex的基本框架

构造函数

非指针类的核心特点在于其数据不是指针,如下面复数模板类的成员数据是T,关键点如下:

  • _Complex_H_防卫式声明
  • 构造函数放在public中,构造函数可以被外部调用
    • 构造函数中有两个阶段完成数据初始化,分别是初始化列表中完成数据初始化和构造函数体内完成数据初始化
    • 该构造函数指定了默认参数,所有此外还定义Complex() : re(0), im(0) {}这样的函数,就会导致编译器不知道调用哪个不带参数的函数
    • 构造函数不能直接调用,只能在对象创建时被间接调用
      • 1、learn::Complex<double> c1(2.5, 1.5);调用构造函数1
      • 2、learn::Complex<double> c2;调用构造函数1,使用默认参数
      • 3、learn::Complex<double> c3();这不是定义一个Complex对象是声明了一个函数,函数名为c3
      • 4、learn::Complex<double> *p = new learn::Complex<double>(4);// 定义一个Complex对象,让指针p指向该对象
    • 当没有调用拷贝构造、拷贝赋值、析构函数时,编译器会生成一份拷贝构造、拷贝赋值、析构函数,这份拷贝构造和拷贝赋值就是就是简单的对象内存拷贝,而这份析构函数没有做如何处理,这对非指针类型的Class来说刚好够用,像Complex类其数据就是re和im;而指针类的class必须自己实现这三个函数,不实现拷贝构造、拷贝赋值会导致浅拷贝,不实现析构函数可能会造成内存泄露
  • 数据放在private作用域中起到对数据封装的作用
#ifndef _Complex_H_
#define _Complex_H_
#include <ostream>
#include <cmath>

namespace learn
{
	// 模板类声明格式
    template <class T>
    class Complex;
    // ... 此次可以申明一些用到Complex的全局函数

    template <typename T>
    class Complex
    {
    public:
        // 构造函数1-指定默认实参
        Complex(T r = 0, T i = 0) : re(r), im(i)
        {
			std::cout << "use default ctor " << "r: "<< r << ", i:"<< i << std::endl;
        }
    private:
        T re, im;
    };
} // end namespace learn
#endif

构造函数不一定放在public中,经典的单例模式实现如下,其构造函数就放在private中;

  • 函数中定义的static变量对应的内存在堆中,第一次调用getInstance时定义该变量(分配内存),直到程序退出时才死亡; 不调用getInstance函数不定义定义对象,这样节省了内存;
  • 构造函数都被定义在private中,在类的外部无法直接定义,只能通过getInstance函数创建一份A对象
  • static函数是为static成员变量服务
namespace learn
{
    class A;

    class A
    {
    public:
        static A &getInstance();
        void setup() {}

    private:
        A() {}
        A(const A &rhs) {}
    };

	A &A::getInstance()	
    {
        static A a;
        return a;
    }
}


this指针

c++成员函数的参数都带有一个this指针,该指针指向对象本身,因此可以通过this指针来访问/修改对象的成员数据

  • 成员函数参数与函数体之间的const用来修饰指针this,即const typename *this;因此这类函数不能修改入参this指向的对象
  • 如果一个const对象调用了非const方法,编译器会报passing ‘const learn::Complex<int>’ as ‘this’ argument discards qualifiers [-fpermissive]将const learn::Complex<int>传递给this指针会丢失掉const限定符
  • 比如a.fun(b)的调用方式,为什么对象a可以在func中直接拿到b的私有数据呢？ => 相同class的各对象互为friends
    template <typename T>
class Complex
{
    // const表示不能通过this来修改对象的数据
    T real() const { return re; }
    T imag() const { return im; }

    // this 对象直接访问 另外一个(同类型)对象的private数据, 相同class的各对象互为friends
    T func(const Complex<T> &param) const
    {
        return param.re + param.im;
    }
};


友元函数

友元函数声明告诉编译器__doapl是我这个类的友元, 那么可以在该函数中直接访问该类的private数据或方法,这也打开了封装的大门

  • 友元模板函数必须定义在.h文件中
  • inline和模板函数的格式
  • __doapl是do assign plus执行赋值加法
    • ths指针会被修改, 所以没有被声明为const
    • r对象不会被修改, 所以声明为const
// 模板类声明格式
template <class T>
class Complex;

template <typename T>
Complex<T> &__doapl(Complex<T> *ths, const Complex<T> &r);

template <typename T>
class Complex
{
    public:
    // ...
    // 成员函数重载
    Complex<T> &operator+=(const Complex<T> &rhs){return __doapl(this, rhs);}

    private:
        T re, im;

    // 友元函数声明
    template <typename U>
    friend Complex<U> &__doapl(Complex<U> *, const Complex<U> &);
};

template <typename T>
inline Complex<T> &__doapl(Complex<T> *ths, const Complex<T> &r)
{
    ths->re += r.re;
    ths->im += r.im;
    return *ths; // 传递着无需知道接收者以reference形式接收
}

操作符重载

二元操作符重载

  • [同一个class] 当我们想执行类似object1 += object2这样的运算时, 编译器会去在class中找一个函数function来完成object2.function(object2);所以function是class的成员函数, 按照C++的语法,这个function应该被定义为:
    • 该函数的入参是object2,且是const Complex<T> &
    • 该运算的结果还是object1(也即该函数的返回值)

Complex<T> &operator+=(const Complex<T> &rhs){return __doapl(this, rhs);}

  • [不同class] 当object1和object2不是同一个class时,且object1的class不是自己实现的类,比如std::cout << object1; 此时的二元操作符重载应该按照如下实现
    • 将该函数声明为为Complex的友元函数, 方便取数据
    • 输出时会改变os,所以没有用const修饰os; 不改变rhs,所以使用const修改rhs
    • 该函数返回os是为了方便连续输出, 比如std::cout << object1 << "::" << std::endl;
    • os在调用<<之前就已经存在, 所以返回引用没有问题
    • 实数 + 复数的案例中, 使用typename() 创建临时对象(temp object),所以返回时必须使用reture by value
    • <<必须定义为函数重载;复数 + 实数可以是成员函数重载也可以是全局函数重载
template <typename T>
class Complex
{
    public:
    // ...
    // 复数 + 实数
    Complex<T> operator+(double re) const
    {
        return Complex<T>(this->re + re, this->im); // 这里会创建一个临时变量, 所以返回值是return by value
    }

    private:
        T re, im;

    // 友元函数声明
    template <typename U>
    friend std::ostream &operator<<(std::ostream &os, const Complex<U> &rhs);

    // 实数 + 复数
    template <typename U>
    friend Complex<U> operator+(double re, const Complex<U> &rhs);

    // 复数 + 实数
    template <typename U>
    friend Complex<U> operator+(const Complex<U>& lhs, double re) ; 
};

template <typename U>
inline std::ostream &operator<<(std::ostream &os, const learn::Complex<U> &rhs)
{
    os << "(" << rhs.re << ", " << rhs.im << "i)";
    return os;
}

// 实数 + 复数
template <typename U>
Complex<U> operator+(double re, const Complex<U> &rhs)
{
    return Complex<U>(re + rhs.re, rhs.im);
}

// 复数 + 实数
template <typename U>
Complex<U> operator+(const Complex<U>& lhs, double re)
{   
    std::cout << "global function" << std::endl;
    return Complex<U>(lhs.re + re, lhs.im);
}

一元操作符重载

template <typename T>
class Complex
{
    public:
    // ...
    private:
        T re, im;

    // 友元函数声明
    // 取反
    template <typename U>
    friend Complex<U> operator-(const Complex<U> &rhs);
};

template <typename U>
Complex<U> operator-(const Complex<U> &rhs)
{
    return Complex<U>(-rhs.re, -rhs.im);
}
