freeRTOS-01-任务引入_ARM架构简明教程

发表于2022-05-12|更新于2022-05-14|FreeRTOS
|字数总计:1.5k|阅读时长:5分钟|阅读量:

什么是任务
任务是运行起来的函数

  • 一段代码:保存在flash上(不仅仅是代码)
  • 运行位置:
  • 运行的环境

要理解任务,先了解ARM架构

ARM架构

常用汇编指令

  • 读-Load:

    • LDR -读4个字节: LDR R0, [addr] 从地址addr读到R0
    • LDRH-读2个字节: LDRH R0, [addr]
    • LDRB-读1个字节: LDRB R0, [addr]
    • POP -读指令:pop {R3,PC}

  • 写-Store:

    • STR-写:STR R0, [addr] 把R0的值写入地址addr

  • 计算:

    • ADD:ADD R0,R1,R2 - R0 = R1 + R2
    • SUB:SUB R1,R1,R2 - R1 = R1 - R2

  • 写内存指令-push:

    • push:push {R3, LR}-按照栈sp给定的地址写内存
  • 赋值指令-movs:
    • movs:movs r3 #0-把0赋值给r3

push {R3, LR}过程:

  • 把r14即lr寄存器的值写入sp，同时执行sp=sp-4
  • 把r3寄存器的值写入sp，同时执行sp=sp-4

    低标号的寄存器写入栈的低地址,所以push后的花括号内的顺序无所谓;这里lr是r14

pop {R3, pc}过程,该过程与push过程相反:

  • r3=[sp],同时sp=sp+4
  • pc=[sp],同时sp=sp+4

push和pop的流程刚好与栈的概念相符合:

  • 栈区的使用是从高地址到低地址
  • 栈区的使用遵循先进后出,后进先出
  • 栈区的放置是从高地址往低地址放置(push压栈)
  • 删除是从低往高删除(pop出栈)

STM32寄存器

STM32的r寄存器有15个,如图STM32寄存器所示,其中:

  • r13的别名为sp(栈)
  • r14的别名为lr(返回地址)
  • r15的别名为pc(当前指令地址)

a = a + b流程

freeRTOS中使用ucheap巨大的数组来当做栈使用
被定义在Middlewares\Third_Party\FreeRTOS\Source\portable\MemMang\heap_4.c

#define configTOTAL_HEAP_SIZE                    ((size_t)3072)
static uint8_t ucHeap[ configTOTAL_HEAP_SIZE ];

由build\09-FreeRTOS_V1_DEMO.map可知,栈ucHeap的起始地址为0x20000464,而下一个segment-.bss.xBlockAllocatedBit的起始地址为0x20001064;计算可知0x20001064-0x20000464 = 0xc00 = 3072,这与代码中分配的内存空间大小相符合;
.bss.pxEnd     0x20000460        0x4 build/heap_4.o
.bss.ucHeap    0x20000464      0xc00 build/heap_4.o
.bss.xBlockAllocatedBit
               0x20001064        0x4 build/heap_4.o

以如下函数的执行过程描述汇编指令的运行过程

/* USER CODE END Header_LED0_Turnover_run */
void LED0_Turnover_run(void const *argument)
{
  /* USER CODE BEGIN LED0_Turnover_run */
  volatile uint16_t counterLT = 0;
  int a = 1, b = 1;

  /* Infinite loop */
  for (;;)
  {
    add_val(&a, &b);
    counterLT++;
    osDelay(400);
  }
  /* USER CODE END LED0_Turnover_run */
}

/* Private application code --------------------------------------------------*/
/* USER CODE BEGIN Application */
void add_val(int *pa, int *pb)
{
  volatile int temp;
  temp = *pa;
  temp += *pb;
  *pa = temp;
}
/* USER CODE END Application */

STM32内部的ARM框架如下图所示:

以下内容不是非常严谨(地址和value有混淆的地方),但大致流程是对的;

breakpoint 1

  • 由上图可知,寄存器sp为0x200008d0 <ucHeap+1132>
  • 在汇编指令0x0800 03ce中将返回值寄存器lr压入sp(同时sp=sp-4,此时sp = 0x200008cc <ucHeap+1128>)
  • 在汇编指令0x0800 03d0中,sp = sp - #20,此时sp = 0x200008b8 <ucHeap+1108>

breakpoint 2

  • 在汇编指令0x0800 03d2中,给寄存器r3赋0
  • 在汇编指令0x0800 03d4中,把寄存器r3赋值到栈[sp + 14],这应该是把counterLT放在栈[sp + 14]开始的2个字节,同时赋初值0;所以counterLT的地址为0x200008b8 + #14 = 0x200008c6 <ucHeap+1122>

breakpoint 3

  • 在汇编指令0x0800 03d8中,给寄存器r3赋1
  • 在汇编指令0x0800 03da中,把寄存器r3写入栈[sp + #8],即把变量a放在[sp + #8] = 0x200008c0 <ucHeap+1116>
  • 在汇编指令0x0800 03dc中,把寄存器r3写入栈[sp + #4],即把变量a放在[sp + #8] = 0x200008bc <ucHeap+1112>

breakpoint 4

  • 在汇编指令0x0800 03de中,把栈[sp + #4](变量b的地址)写入寄存器r1
  • 在汇编指令0x0800 03e0中,把栈[sp + #8](变量a的地址)写入寄存器r0
  • 在汇编指令0x0800 03e2中,使用bl跳转到函数add_val,并把0x800,03E7保存在返回寄存器lr

在调用c函数时,默认第一个参数a保存在r0,第二个参数b保存在r1

breakpoint 5

  • 在汇编指令0x0800 03b8中,把寄存器sp自减为sp = sp - #8 = 0x200008b0 <ucHeap+1100>

breakpoint 6

  • 在汇编指令0x0800 03ba中,把寄存器r0(变量a的地址)加载到r3中
  • 在汇编指令0x0800 03bc中,把寄存器r3(变量a的地址)写入栈[ sp + #4](地址对应栈的数据),相当于temp = *pa,temp的栈地址为[ sp + #4] = 0x200008b4 <ucHeap+1104>

breakpoint 7

  • 在汇编指令0x0800 03be中,加载r1(变量b的地址)到r3
  • 在汇编指令0x0800 03c0中,加载[ sp + #4](变量temp)到r2
  • 在汇编指令0x0800 03c2中,r3 = r3 + temp = b + temp = b + a
  • 在汇编指令0x0800 03c4中,把r3写入[ sp + #4 ],即写入栈内的temp

breakpoint 8

  • 在汇编指令0x0800 03c6中,把栈内[ sp + #4 ](变量temp)读入到r3
  • 在汇编指令0x0800 03c8中,把r3写入r0 + #0(变量a的地址)

breakpoint 9

  • 在汇编指令0x0800 03ca中, 将sp = sp + #8 = 0x200008b8 <ucHeap+1108>
  • 在汇编指令0x0800 03cc中,跳转到lr(返回值-执行add_val)

函数运行环境

从汇编的角度理解c函数的运行过程后,如果有以下过程:

  • 执行0x0800 03ba,从r0指向的栈内取值-读变量a
  • 被打断, 0x0800 03bc没能被执行,则temp = *pa没被执行完成
    此时如何保存现场:此时应该保存r0的值

什么是现场

现场就是被中断时CPU内部的寄存器r0-r15和状态寄存器的值

怎么保存

把CPU内部的寄存器r0-r15和状态寄存器的值保存到该任务的栈内

所以任务=函数+现场(栈),即运行中的函数

保存现场的几种场景

硬件中断

stm32的硬件中断,当发硬件中断时,同样要保存当前的现场,处理中断,然后恢复中断;
对应F103,Cortex,M3,M4,保存逻辑如下:

  • 硬件保存一部分:R0,R1,R2
  • 软件保存一部分:一些用到的寄存器

函数调用

funA调用funB,同样要保存当前的现场,处理funB,然后恢复中断;

void funA()
{
  ...
  funB();
  ...
}

函数调用中r0,r1,r2被用来传递参数,保存现场时不需要保存

任务切换

任务切换时,CPU的所有寄存器都会被保存
