ps

ps命令用于显示当前进程的状态，类似于windows的任务管理器,全称为process status。

ps -ef

  • 参数说明

    • e 等同于参数A,查看所有进程
    • f full-format,以表格的形式展示

  • 示例

    ➜  build ps -ef | grep test_top
tangshp    20460   17304  0 14:41 pts/2    00:00:00 ./test_top
tangshp    20504   11929  0 14:41 pts/0    00:00:00 grep --color=auto test_top
    • tangshp是启动该程序的用户
    • 20460是该程序的进程ID
    • 17304是该程序的夫进程ID, 父进程ID小于子进程ID
    • 0是CPU占用率
    • 14:41是程序的启动时间
    • pts/2表示运行在哪个终端(虚拟终端); 如果是?,表示与终端无关;
    • 00:00:00是使用了多长时间CPU
    • ./test_top是应用程序的名字

ps -aux

ps -aux使用BSD风格显示进程状态

  • 示例
    ➜  build ps -aux | grep test_top
tangshp    20460  0.1  0.0  10320   508 pts/2    Sl+  14:41   0:02 ./test_top
tangshp    21058  0.0  0.0   9048   756 pts/0    S+   15:11   0:00 grep --color=auto test_top
➜  build ps -aux | head -n 1
USER         PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
    • VSZ表示使用的虚拟内存(Kbytes)
    • RSS表示使用的固定内存(Kbytes)
    • STAT表示进程的状态; 查看man ps中的PROCESS STATE CODES可知
      D    uninterruptible sleep (usually IO) 不可终端状态
I    Idle kernel thread
R    running or runnable (on run queue) 目前正在运行或者在运行队列中
S    interruptible sleep (waiting for an event to complete) 表示该程序正在睡眠中,但可以被某些信号唤醒
T    stopped by job control signal 目前该程序暂停了,转到后台运行
t    stopped by debugger during the tracing
W    paging (not valid since the 2.6.xx kernel)
X    dead (should never be seen)
Z    defunct ("zombie") process, terminated but not reaped by its parent 僵尸进程(该进程已经中止,但父进程却无法正常的中止它)
Others-------------
<    high-priority (not nice to other users) 高优先级
N    low-priority (nice to other users) 低优先级
L    has pages locked into memory (for real-time and custom IO) 
s    is a session leader 有子进程
l    is multi-threaded (using CLONE_THREAD, like NPTL pthreads do) 多线程进程
+    is in the foreground process group 前台运行

      Ctrl+z是发送SIGSTOP信号,挂起一个进程, 使其暂停;转到后台运行
      Ctrl+c是发送SIGINT信号,中止一个进程, 使其停止

僵尸进程:僵尸进程是当子进程比父进程先结束,而父进程又没有回收子进程,没有释放子进程占用的资源,此时子进程将成为一个僵尸进程。如果父进程先退出,子进程被init接管,子进程退出后init会回收其占用的相关资源。

top

说明

动态查看进程状态(可实时更新进程状态)

top - 15:45:18 up  6:37,  3 users,  load average: 0.10, 0.10, 0.09
Tasks: 186 total,   1 running, 183 sleeping,   2 stopped,   0 zombie
%Cpu(s):  0.2 us,  0.3 sy,  0.0 ni, 99.5 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
MiB Mem :   7759.4 total,   5449.4 free,    580.4 used,   1729.6 buff/cache
MiB Swap:      0.0 total,      0.0 free,      0.0 used.   7064.9 avail Mem

  • 第一行和uptime的结果一模一样
  • Tasks: 186 total, 1 running, 183 sleeping, 2 stopped, 0 zombie分别表示进程的数量, 运行进程的数量, 睡眠的进程的数量, 暂停的进程的数量, 僵尸进程的数量;
  • 在使用top查看动态进程状态时,按下1时, 第3行会变成如下(显示每个核的具体信息, 这里可知为什么CPU使用率会超过100%, 多核的原因); 
    %Cpu0  :  0.3 us,  0.3 sy,  0.0 ni, 99.3 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
%Cpu1  :  0.0 us,  0.0 sy,  0.0 ni,100.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
%Cpu2  :  0.0 us,  1.0 sy,  0.0 ni, 99.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
%Cpu3  :  0.0 us,  0.0 sy,  0.0 ni,100.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
    • us 用户进程使用CPU的百分比
    • sy 内核进程使用CPU的百分比
    • ni 用户空间内改变过优先级的进程占用CPU的百分比
    • id 空闲CPU的百分比
  • MiB Mem : 7759.4 total, 5449.4 free, 580.4 used, 1729.6 buff/cache分别表示物理内存的总量,空闲内存的总量,已使用内存的总量,缓存的总量

    后三项之和等于第一项

快捷键

  • 默认是3s的刷新周期, 使用space手动刷新
  • 使用q退出
  • 使用M对内存排序
  • 使用P对CPU排序
  • 使用 < 或 > 翻页

decorate.cc

// decorate.cc
#include <stdio.h>

int func(int i){
    printf("int func(int i)\n");
    return 0;
}

float func(float i){
    printf("float func(float i)\n");
    return 0;
}

class C{
public:
    int func(int i){
        printf("float C::func(int i)\n");
        return 0;
    }
    class C2{
    public:
        int func(int i){
            printf("int C2::C::func(int i)\n");
            return 0;
        }
    };
};

namespace N{
    int func(int i){
        printf("int N::func(int i)\n");
        return 0;
    }
    class C{
    public:
        int func(int i){
            printf("int N::C::func(int i)\n");
            return 0;
        }
    };
}

int main(int argc, char **argv){
    C c;
    c.func(1);

    C::C2 c2;
    c2.func(1);

    N::C nc;
    nc.func(1);
    return 0;
}

  1. 查看进程状态

    ➜ Blog pm2 list

  2. 重启运用

    pm2 restart app.js

  3. 停止特定的应用。可以先通过pm2 list获取应用的名字（—name指定的）或者进程id

    pm2 stop app_name|app_id

  4. 停止所有应用

    pm2 stop all

npm

本地与全局

npm的包安装分为本地安装local、全局安装global两种，从敲的命令行来看，差别只是有没有-g而已，比如

npm install grunt # 本地安装
npm install -g grunt-cli # 全局安装

主要区别在于:

  • 本地安装
    将安装包放在./node_modules下（运行npm时所在的目录） 可以通过require()来引入本地安装的包
  • 全局安装
    将安装包放在/usr/local下 可以直接在命令行里使用

—save

  • 在npm 5之前的版本
    使用npm install默认选项安装包时，仅仅会把包下载到node_modules/中，并不会同时修改 package.json。而使用--save选项就可以在安装包的同时，修改package.json文件
  • 在npm 5之后的版本
    npm install安装包时，默认便会修改package.json文件，所以--save选项已经不再需要了。
