生成linux-4.9.229内核镜像

源码编译linux-4.9.229内核

环境说明

ubuntu 18.04
linux-4.9.229
busybox-1.30.0
qemu-system-x86_64

编译过程说明

1.下载linux-4.9.229.tar.gz并解压，进入linux-4.9.229

2.`linux-4.9.229/arch/‘中包含很多框架的代码, 为了减少安装编译器的过程，我在本例中使用的是x86

➜  linux-4.9.229 export ARCH=x86

3.生成默认编译选项

➜  linux-4.9.229 make x86_64_defconfig

默认配置文件被写入.config中。

4.自定义编译选项, 配置内核
这一步其实是对第2步的菜单进行微调，我们需要内核支持ramdisk驱动，所以需要选中如下配置：

➜  linux-4.9.229 make menuconfig

执行该命令后可能会报fatal error: curses.h: 没有那个文件或目录的错误, 则应该安装libncurses5-dev

➜  ~ sudo apt-get install libncurses5-dev

安如下配置：

General setup  --->

       ----> [*] Initial RAM filesystem and RAM disk (initramfs/initrd) support

    Device Drivers  --->

       [*] Block devices  --->

               <*>   RAM block device support

               (65536) Default RAM disk size (kbytes)

  1. 编译内核
    make

    编译成功后的内核在arch/x86_64/boot/bzImage

一个最小的busybox文件系统

编译

1.下载busybox-1.30.0.tar.bz2

# tar xvf busybox-1.30.0.tar.bz2

2.配置busybox源码

在这里我们把busybox配置为静态编译，这样busybox在运行的时候就不需要额外的动态链接库了。

# make menuconfig

Busybox Settings  --->
      Build Options  --->
            [*] Build BusyBox as a static binary (no shared libs)

3.编译安装

make && make install

不用担心install会安装到系统的文件中，没有加sudo权限；编译完成后的busybox就安装在源码根目录下的_install目录了; _install/usr/bin内中任意一个文件file都是链接到busybox这么一个可执行文件，运行时，file是busybox的参数

➜  _install ls
bin  linuxrc  sbin  usr
  • _install下的文件不是一个完整的文件系统，需要完善；内核不能直接使用这些文件，所以完善后也不能直接使用；需要需要把这些文件打包成内核可识别的文件系统。

补充一些必要的文件

# mkdir etc dev mnt

# mkdir -p proc sys tmp mnt 

# mkdir -p etc/init.d/ # 该文件夹存放一些开机启动的脚本

# vim etc/fstab # 存放一些挂载的文件信息
proc        /proc           proc         defaults        0        0
tmpfs       /tmp            tmpfs    　　defaults        0        0
sysfs       /sys            sysfs        defaults        0        0
# 把proc虚拟文件系统挂载在/proc下
# 把临时文件系统挂载在/tmp下
# sysfs文件系统挂载在/sys下

# vim etc/init.d/rcS # 启动要运行的文件
echo -e "Welcome to tinyLinux"
/bin/mount -a # 把etc/fstab下的要挂载的文件挂载
echo -e "Remounting the root filesystem"
mount  -o  remount,rw  / # 重新挂载根文件系统, 使得跟文件系统可读可写
mkdir -p /dev/pts
mount -t devpts devpts /dev/pts # 挂载文件系统
echo /sbin/mdev > /proc/sys/kernel/hotplug # 处理热插拔
mdev -s

# chmod 755 etc/init.d/rcS #busybox在启动运行的一段代码

# vim etc/inittab # busybox的一些启动文件
::sysinit:/etc/init.d/rcS # 调用rcS文件运行
::respawn:-/bin/sh # 使得可以在控制台使用命令
::askfirst:-/bin/sh
::ctrlaltdel:/bin/umount -a -r

# chmod 755 etc/inittab
# cd dev
# mknod console c 5 1
# mknod null c 1 3
# mknod tty1 c 4 1

这样一个最小的、完整的可以被内核启动的文件系统就齐活啦。

2.制作根文件系统镜像文件

思路
1.先制作一个空的镜像文件；
2.然后把此镜像文件格式化为ext3格式；
3.然后把此镜像文件挂载，并把根文件系统复制到挂载目录；
4.卸载该镜像文件。
5.打成gzip包。
打包脚本如下：

#!/bin/bash
rm -rf rootfs.ext3
rm -rf fs
dd if=/dev/zero of=./rootfs.ext3 bs=1M count=32
mkfs.ext3 rootfs.ext3
mkdir fs
mount -o loop rootfs.ext3 ./fs
cp -rf ./_install/* ./fs
umount ./fs
gzip --best -c rootfs.ext3 > rootfs.img.gz

最终生成的文件系统镜像名字为：rootfs.img.gz

在ubuntu20.04中, 树莓派4B+8G的软硬件环境编译busybox

#----------------环境配置
#       aarch64-linux-gnu-g++
sudo apt install -y g++ g++-aarch64-linux-gnu 
#       menuconfig
sudo apt install -y libncurses5-dev flex bison

#----------------编译过程
#下载源码
wget https://busybox.net/downloads/busybox-1.31.1.tar.bz2
tar -jxvf busybox-1.31.1.tar.bz2
cd busybox-1.31.1

#生成默认编译配置    生成文件大小约2.04MB。
make defconfig    

#编辑配置
make menuconfig
#Settings --->
#    --- Build Options
#    [ ] Build static binary (no shared libs)   #按“空格键”勾选此项。

make -j8 ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu-    #-j8代表8线程编译

qemu模拟器启动

1.安装模拟器

➜  ~ sudo apt-get install qemu-system-x86

2.通过qemu模拟器启动我们自己编译的内核和文件系统：

# qemu-system-x86_64 \
  -kernel ./linux-4.9.229/arch/x86_64/boot/bzImage  \
  -initrd ./busybox-1.30.0/rootfs.img.gz   \
  -append "root=/dev/ram init=/linuxrc"  \
  -serial file:output.txt

在树莓派+buntu20.04编译

编译Linux内核

编译流程与前述类似, 不同之处在以下; 同时编译得到的内核位于Linux_ws/linux-4.9.229/arch/arm64/boot/Image

export ARCH=arm64
export CROSS_COMPILE=aarch64-linux-gnu-

编译busybox文件系统

1.编译流程与前述类似, 不同之处在需修改busybox-1.30.0/util-linux/rdata.c,添加stime函数定义;

int stime(const time_t *t)
{
        struct timeval tv;
        tv.tv_sec = *t;
        tv.tv_usec = 0;
        return settimeofday(&tv, NULL);
}

2.修改fstab文件

➜  busybox-1.30.0 cat ./_install/etc/fstab
proc        /proc           proc         defaults        0        0
sysfs       /sys            sysfs        defaults        0        0
#tmpfs       /tmp            tmpfs    　　defaults        0        0

3.修改打包方式

➜  _install pwd
/home/tangshp/Tools/Linux_ws/busybox-1.30.0/_install
➜  _install find . | cpio -o --format=newc > rootfs.img
➜  busybox-1.30.0 cp ./_install/rootfs.img ./

使用模拟器启动

➜  Linux_ws ls
busybox-1.30.0  linux-4.9.229
➜  Linux_ws sudo qemu-system-aarch64 -cpu cortex-a72 -machine type=virt -nographic -smp 8 -m 1024M -kernel ./linux-4.9.229/arch/arm64/boot/Image -append "root=/dev/ram rdinit=/linuxrc" -initrd ./busybox-1.30.0/rootfs.img -device virtio-scsi-device

    按下ctrl+a 后松开;按下x
