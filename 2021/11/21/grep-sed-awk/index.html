shell编程

debug

  • 以debug模式运行脚本

1.带+表示执行过程
2.没有+表示标准输出

  • 在脚本中指定一部分脚本开启debug

1.set -x开启debug
2.set +x关闭debug

正则

正则符号

  • 基础正则( ^ $ ^$ . * .* [a-z] [^abc] )-(Basic RE-BRE)
    支持的命令包括grep/sed/awk
  • 扩展正则( + | () {} ?)-(Extended RE-ERE)
    支持的命令包括egrep/sed -r/awk

正则VS通配符

分类 用途 支持的命令
正则(re) 三剑客,高级语言;被用作过滤(匹配字符) 三剑客grep,sed,awk,find,rename,expr
通配符(pathname extension 或则glob) 匹配文件名(*.txt, *.log) Linux中大部分命令

基础正则

素材
grep_help.txt

  • ^xxx-以xxx开头的行
    搜索以In开头的行

  • xxx$-以xxx结尾的行
    搜索以.结尾的行

  • ^$-空行
    空行表示没有任何字符(空格也没有)

    用法:排除空行

  • .-任意一个字符
    但是会忽略空行

  • x*-0次及0次以上出现x

  • .*-所有
    运用:匹配以指定字符结束的行

    到最后一个指定字符, 则体现了正则表达式的贪婪行

  • [abc]-一次匹配一个字符(a,b,c中任意一个)

    • 匹配大小写与数字

    • []中添加其它的符号
      除了匹配a-Z 0-9以外,还要匹配,和.

  • [^xxx]-匹配除xxx以外的内容

扩展正则

  • +-匹配1或则1个以上的字符,并输出匹配到的行
    要使用扩展正则命令,需要以以下三种方式书写

    • 匹配文件中连续的数字
      素材ascii.txt

      -o表示输出过程,可获得匹配到的对象,而不是整行

  • |-或者

匹配单词或则数值


符号 区别 运用场景
[] 1一次匹配一个字符[abc] 匹配单个字符
| a | b | c | string 匹配单词

  • ()-括号

    1.被括号括起来的内容是一个整体
    2.反向引用-sed

    • 匹配tooth或者teeth
      素材bracket.txt

  • {}-连续出现
    使用格式为,
    | 格式 | 说明 | 备注 |
    |:—————————————————-:|:——————————————————:|:——————————————————-:|
    | x{n,m} | 表示前一个字母x至少连续出现n次,之多出现m次 |
    | x{n}| 表示前一个字母x恰好连续出现n次 | |
