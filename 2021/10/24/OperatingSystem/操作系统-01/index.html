  • 使用readelf解析elf文件

    ELF Header:
  Magic:   7f 45 4c 46 02 01 01 00 00 00 00 00 00 00 00 00
  Class:                             ELF64
  Data:                              2's complement, little endian
  Version:                           1 (current)
  OS/ABI:                            UNIX - System V
  ABI Version:                       0
  Type:                              DYN (Shared object file)
  Machine:                           AArch64
  Version:                           0x1
  Entry point address:               0x610
  Start of program headers:          64 (bytes into file)
  Start of section headers:          7376 (bytes into file)
  Flags:                             0x0
  Size of this header:               64 (bytes)
  Size of program headers:           56 (bytes)
  Number of program headers:         9
  Size of section headers:           64 (bytes)
  Number of section headers:         28
  Section header string table index: 27
关键字 说明
ELF Header 指名ELF文件头开始
Magic 用来指名该文件是一个ELF目标文件。第一个字节7F是个固定的数；后面的3个字节正是E, L, F三个字母的ASCII形式。
CLASS 表示文件类型，这里是64位的ELF格式
Data 表示文件中的数据是按照什么格式组织的(大端或小端)，不同处理器平台数据组织格式可能就不同，如x86平台为小端存储格式
Version ELF文件头版本号，这里版本号为1
OS/ABI 指出操作系统类型，ABI是Application Binary Interface的缩写
ABI Version ABI版本号，当前为0
Type 文件类型。ELF文件有3种类型，一种是如上所示的Executable可执行文件， 一种是可重定位目标文件(Relocatable file)，另外一种是共享库(Shared Library)
Machine 机器平台类型, arm 64
Version 当前目标文件的版本号
Entry point address 程序的虚拟地址入口点，这里是0x610
Start of program headers 程序头开始处，这里是64个字节,表示偏移量
Start of section headers section头开始处，这里7376是十进制,表示偏移量
Flags 一个与处理器相关联的标志
Size of this header ELF文件头的字节数，这里为64
Size of program headers 程序头字节数，这里为56
Number of program headers 程序头数量，这里为9个
Size of section headers sections header的大小，这里每个section头大小为64个字节
Number of section headers 一共有多少个section头，这里是28个
Section header string table index section头字符串表索引号，从Section Headers输出部分可以看到其内容的偏移在64个字节处，从此处开始到0x50结束保存着各个sections的名字，如.data，.text，.bss等。

    上表Section Headers中的各字段,由头文件/usr/include/elf.h中结构体Elf64_Shdr定义
    typedef struct
{
  Elf64_Word    sh_name;        /* Section name (string tbl index) */
  Elf64_Word    sh_type;        /* Section type */
  Elf64_Xword   sh_flags;       /* Section flags */
  Elf64_Addr    sh_addr;        /* Section virtual addr at execution */
  Elf64_Off sh_offset;      /* Section file offset */
  Elf64_Xword   sh_size;        /* Section size in bytes */
  Elf64_Word    sh_link;        /* Link to another section */
  Elf64_Word    sh_info;        /* Additional section information */
  Elf64_Xword   sh_addralign;       /* Section alignment */
  Elf64_Xword   sh_entsize;     /* Entry size if section holds table */
} Elf64_Shdr;

最小的hello world

失败的尝试#1

➜  P2 cat ./hello.c
#include <stdio.h>

int main(int agrc, char **argv)
{
        printf("hello world!\n");
        return 0;
}
  • 针对以上代码进行编译
    • 编译 gcc -c hello.c -o hello.o
    • 链接ld hello.o
      链接时报错, 如下所示
      ➜  P2 gcc -c ./hello.c -o hello.o
➜  P2 ld ./hello.o
ld: warning: cannot find entry symbol _start; defaulting to 00000000004000b0
ld: ./hello.o: in function `main':
hello.c:(.text+0x18): undefined reference to `puts'

      1.尽管使用gcc -oo,编译器也会把printf优化为puts函数(该优化有一点性能提升);因为没有链接libc,所以找不到puts函数;但是链接libc就不是最小的hello world了
      2.main函数不是程序在二进制意义上的入口, _start是连接器默认的入口; 可以使用-e main指定入口函数

失败的尝试#2

为了不链接glibc, 不调用printf,改为如下代码:

➜  P2 cat ./hello.c
int main(int agrc, char **argv)
{
        return 0;
}

重新编译:
➜  P2 gcc -c -o0  ./hello.c -o hello.o
➜  P2 ld -e main ./hello.o

运行
➜  P2 ./a.out
[1]    24658 segmentation fault (core dumped)  ./a.out
