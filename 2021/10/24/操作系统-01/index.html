关键字 说明
ELF Header 指名ELF文件头开始
Magic 用来指名该文件是一个ELF目标文件。第一个字节7F是个固定的数；后面的3个字节正是E, L, F三个字母的ASCII形式。
CLASS 表示文件类型，这里是64位的ELF格式
Data 表示文件中的数据是按照什么格式组织的(大端或小端)，不同处理器平台数据组织格式可能就不同，如x86平台为小端存储格式
Version ELF文件头版本号，这里版本号为1
OS/ABI 指出操作系统类型，ABI是Application Binary Interface的缩写
ABI Version ABI版本号，当前为0
Type 文件类型。ELF文件有3种类型，一种是如上所示的Executable可执行文件， 一种是可重定位目标文件(Relocatable file)，另外一种是共享库(Shared Library)
Machine 机器平台类型, arm 64
Version 当前目标文件的版本号
Entry point address 程序的虚拟地址入口点，这里是0x610
Start of program headers 程序头开始处，这里是64个字节,表示偏移量
Start of section headers section头开始处，这里7376是十进制,表示偏移量
Flags 一个与处理器相关联的标志
Size of this header ELF文件头的字节数，这里为64
Size of program headers 程序头字节数，这里为56
Number of program headers 程序头数量，这里为9个
Size of section headers sections header的大小，这里每个section头大小为64个字节
Number of section headers 一共有多少个section头，这里是28个
Section header string table index section头字符串表索引号，从Section Headers输出部分可以看到其内容的偏移在64个字节处，从此处开始到0x50结束保存着各个sections的名字，如.data，.text，.bss等。
